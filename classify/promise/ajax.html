原生Ajax #

Ajax简介

通过Ajax可以在浏览器中向服务器发送异步请求，最大优势：页面无刷新获取数据

Ajax的特点

优点：

  1. 无需刷新页面与服务器通信
  2. 允许你根据用户事件来更新部分页面内容

缺点：

  1. 没有浏览历史，不能回退
  2. 存在跨域
  3. SEO(搜索引擎优化)不友好

Ajax的使用 #

搭建ajax请求服务器

js
const express = require('express')

const app = express()

//暴露静态资源 __dirname:当前路径
app.use(express.static(__dirname))

// 配置路由
app.get('/test',(req,res) => {
	res.send('ok')
})

app.listen(8080,(err) => {
	(!err) => console('服务器在http://127.0.0.1:8080启动成功')
})

http常用的请求方法

  1. GET：向服务器获取资源
  2. POST：向服务器提交数据，并发送请求
  3. PUT：向服务器更新数据
  4. DELETE：删除指定资源

创建ajax请求

js
// 获取按钮
const btn = document.queryselctor('btn-login')

btn.onclick = () => {
    // 创建xhr实例
	const xhr = new XMLHttpRequest()

	// 监听准备状态改变
	xhr.onreadystatechange = () => {
            if (xhr.readyState == 4) {
               // 状态码在200-300表示响应成功
               if (xhr.status >= 200 && xhr.status < 300) {
                    console.log(xhr.response);
               }
            }
    }

	// 指定发送请求 参数：发送方式，URL
	xhr.open('GET','http://127.0.0.1:8080/test')

	// 发送请求
	xhr.send()
}

POST请求 #

POST请求发送的参数

  1. POST请求的参数一般是body
  2. POST请求也可以实现请求query参数和params参数
  3. body参数可以用urlencoded编码格式或者JSON格式

浏览器发送POST请求

js
// 指定请求方式
xhr.open('POST', 'http://127.0.0.1:8080/test')

// 设置请求头,解析encoded编码格式
xhr.setRequestHeader('Content-type', 'application/x-www-form-urlencoded'
// 设置请求头,解析json编码格式
xhr.setRequestHeader('Content-type', 'application/json'

// 发送urlencoded格式请求
xhr.send('name=zs&age=18')
// 发送json格式请求
xhr.send(JSON.stringify({name:'zs',age:18}))

避免重复发送请求 #

在服务器为响应给浏览器之前，如果用户多次点击发送按钮，我们要把用户前面的所有发送请求都给干掉，保留最后一个请求

js
// 定义一个正在加载的标识符
let isLoading
btn.onclick = () => {
    if (isLoading) xhr.abort()
    ...
    // 响应请求
    if (xhr.status >= 200 && xhr.status < 300) {
   		isLoading = false
    }
    ...
    // 发送请求
    xhr.send()
    isLoading = true
}

Jquery封装Ajax #

浏览器发送GET请求

  1. 完整写法：
js
// 获取按钮
const btn = $('.btn-login')
// 绑定点击事件
btn.click(() => {
    // 完整写法
    $.ajax({
        url:'http://127.0.0.1:8080/test_timeOut',
        method:'GET', //如果不写，默认是GET请求
        data:{ name:'zs' }, //GET请求携带的参数,默认是query参数
        dataType:'json',//配置解析响应数据格式
        success:(result) => {
            console.log(result);
        }, //成功的回调
        error:() => {
            console.log('请求出错'); //失败的回调
        }
    }) 
})

跨域与同源策略 #

为什么会产生跨域

浏览器为了安全，采用同策略

同源

协议（http）、域名或者IP、端口号必须完全相同

不同源受到哪些限制

  1. Cookie不能读取
  2. DOM无法获得
  3. Ajax请求不能获取数据

jsonp解决跨域 #

绕开xhr，借助script标签发请求不受同源策略影响，有种前端定义函数，后端调用函数感觉

cors解决跨域 #

js
// 后端路由配置特殊的响应头
res.setHeader('Access-Control-Allow-Origin','*') // 所有的端口号都能访问

res.setHeader('Access-Control-Allow-Origin','http://127.0.0.1:5050') // 指定的端口号才能访问
